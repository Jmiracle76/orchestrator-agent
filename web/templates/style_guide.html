Buttons

Use semantic <button> elements with type="button" unless submitting forms.

States: hover elevates + saturates; focus-visible uses 2px outline; disabled applies opacity-50 cursor-not-allowed.

Form controls

Label every input, pair hints with aria-describedby, and keep focus rings visible.

Usage patterns & UX specs

  • • Use semantic HTML: buttons for actions, anchors for navigation, labels for inputs.
  • • Responsiveness: collapse multi-column grids to one column below md; keep 24px outer gutters.
  • • Accessibility: every interactive element has a focus-visible outline; provide aria-labels where text is hidden.
  • • Motion: prefer micro transitions (120–180ms) on hover/focus; avoid flashy animations for blocking flows.
  • • Empty states: use a bordered card with iconography, short headline, and a primary button.
  • • Data density: cap line length to ~72 characters; use max-w-2xl on long-form copy.
